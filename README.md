# MedPrice KZ — агрегатор и сравнение цен на медуслуги в Казахстане

Бэкенд-сервис (FastAPI), который **собирает прайс-листы клиник Казахстана**,
приводит разнородные названия услуг к единому справочнику и **сравнивает цены
одной услуги между клиниками** — с фильтром по городу, статистикой и расчётом
экономии.

Это решение кейса «Агрегатор и система сравнения цен на медицинские услуги в
Казахстане». Внутри живёт и смежный кейс «автоматическая обработка прайсов» —
как модуль нормализации/матчинга.

---

## Почему это нетривиально

Данных «из коробки» нет — их пришлось **добыть**. И главная сложность не в
парсинге, а в том, что одна услуга у разных клиник называется по-разному:

| Клиника | Как называется в прайсе | Цена |
|---|---|---|
| KDL Олимп | «Клинический анализ крови (с лейкоцитарной формулой)» | 800 ₸ |
| INVITRO | «Общий анализ крови с лейкоформулой и СОЭ» | 1 790 ₸ |
| Олимп | «Общий анализ крови без СОЭ» | 1 780 ₸ |

Чтобы сравнивать, всё это нужно свести к одной канонической услуге
**«Общий анализ крови (ОАК)»**. Этим занимается движок матчинга.

## Архитектура (3 слоя)

```
[1] ХАРВЕСТ          harvester/discover.py   sitemap 103.kz -> фронтир 9248 клиник
                     harvester/harvest.py    многопоточно качает прайсы -> raw/*.jsonl

[2] НОРМАЛИЗАЦИЯ     app/normalize.py         raw-название -> каноническая услуга
    + ИНГЕСТ         app/ingest.py            матчинг всех позиций -> SQLite

[3] API             app/main.py              поиск / сравнение / статистика (FastAPI)
```

### 1. Источник данных
Портал **103.kz** агрегирует прайс-листы тысяч клиник РК: каждая клиника живёт
на поддомене `<slug>.103.kz`, её прайс — на `/pricing/` по единому HTML-шаблону.
`discover.py` берёт полный список клиник из `sitemap-personals.xml.gz`
(**9248 клиник**), `harvest.py` многопоточно (resumable) выкачивает прайсы.

Цены реальные, со ссылкой на источник (`source_url`) у каждой клиники.

### 2. Движок матчинга (`app/normalize.py`) — ядро
- **Нормализация**: регистр, `ё→е`, пунктуация, пробелы → канонический ключ.
- **Кураторский справочник** (`app/data/canonical.json`): «якорные» услуги
  (анализы, гормоны, онкомаркеры, МРТ/КТ/УЗИ) и их синонимы.
- **Containment-скоринг**: если синоним целиком входит в raw-название
  («тиреотропный гормон» ⊂ «ТТГ (тиреотропный гормон) ультрачувствительный»)
  — это сильный матч, устойчивый к словам-уточнениям.
- **Длинный хвост**: всё, что не попало в справочник, группируется по
  нормализованному ключу — одинаковые услуги у разных клиник всё равно
  сводятся вместе и становятся сравнимыми. Каталог растёт из данных, O(N).

Метод матча и confidence сохраняются у каждой позиции (`curated` /
`curated_fuzzy` / `auto`) — прозрачно и проверяемо.

### 3. API (`app/main.py`)
CORS открыт — фронт можно вынести отдельно (например, на Vercel).

| Метод | Эндпоинт | Назначение |
|---|---|---|
| GET | `/api/stats` | сводка по базе |
| GET | `/api/cities` | города и число клиник |
| GET | `/api/categories` | категории услуг |
| GET | `/api/search?q=&city=` | быстрый поиск + диапазон цен |
| GET | `/api/services?q=&category=&min_clinics=` | каталог канонических услуг |
| GET | `/api/services/{code}/compare?city=` | **сравнение цен по клиникам** + статистика |
| GET | `/api/clinics?city=&q=` | список клиник |
| POST | `/api/ingest/match` | загрузить строки прайса → матч + вердикт по рынку |

## Запуск

```powershell
pip install -r requirements.txt        # зависимости
python harvester/discover.py           # построить фронтир клиник (1 раз)
python harvester/harvest.py            # выкачать прайсы (resumable)
python -m app.ingest                   # собрать БД из сырья
python -m app.report                   # отчёт покрытия
.\run.ps1 serve                        # API -> http://127.0.0.1:8077/docs
```
Или одной командой: `.\run.ps1 all`.

### База данных
Бэкенд работает **только с Postgres** (Supabase) — строка подключения в `DATABASE_URL`.
SQLite больше не поддерживается: дев и прод используют один и тот же источник данных.

**Текущий прод-инстанс:** Supabase-проект `medprice-kz` (регион eu-central-1,
Postgres 17). Все 277K строк залиты. Пулеры Supabase:

| Порт | Пулер | Для чего |
|---|---|---|
| `:6543` | transaction | приложение (FastAPI / Vercel-функция) — `.\run.ps1 serve` |
| `:5432` | session | пакетные скрипты (`ingest` / `geocode` / `enrich`) |

`run.ps1` подставляет нужный порт автоматически.

> Для деплоя на Vercel задай `DATABASE_URL` (transaction pooler `:6543`) в переменных
> окружения проекта — строкой подключения Supabase (Connect → ORM). RLS см. ниже.
> Функция прибита к региону `fra1` (Франкфурт) в `vercel.json` — рядом с БД.

### Кэш и свежесть цен
Цены обновляются раз в сутки (cron 03:00 UTC, `.github/workflows/refresh-2gis.yml`),
поэтому каталог на фронте кэшируется на 24ч (ISR + тег `catalog`). После фонового
обновления данных workflow дёргает `POST /api/revalidate?secret=…` и кэш сбрасывается
точечно. Секреты репозитория для этого: `SITE_URL`, `REVALIDATE_SECRET`
(на Vercel — переменная окружения `REVALIDATE_SECRET`). Без них кэш всё равно
протухает по таймеру.

## Фронтенд (Next.js) — `web/`

UI на **Next.js 16 (App Router) + TypeScript + Tailwind**, ходит в FastAPI.

| Страница | Что делает |
|---|---|
| `/` | поиск, статистика, категории, топ услуг |
| `/search?q=&city=` | результаты с диапазоном цен / обзор по категории |
| `/service/[code]?city=` | сравнение по клиникам: статистика, экономия, таблица (дешёвые сверху) |
| `/price-check` | вставить строки прайса → распознавание + вердикт по рынку |

```powershell
cd web
npm install
# .env.local: NEXT_PUBLIC_API_BASE=http://127.0.0.1:8077
npm run dev          # -> http://localhost:3000
```

### Деплой на Vercel
1. Фронт (`web/`) — Root Directory `web`, env `NEXT_PUBLIC_API_BASE` = URL задеплоенного API.
2. Бэкенд — FastAPI на Vercel (Python) / Render / Railway; env `DATABASE_URL` = строка
   подключения Supabase. БД уже поднята (проект `medprice-kz`).

## Данные и оговорки
- Источник цен — публичные прайс-листы клиник на портале **103.kz**;
  у каждой клиники сохранён `source_url`. Цены актуальны на момент сбора и
  могут отличаться от фактических — для production нужен регулярный
  переобход (харвестер докачиваемый и для этого готов).
- Часть позиций — «уточняйте» или «от N ₸»; такие помечены флагами
  (`on_request`, `is_from`) и в сравнении по цене не участвуют либо помечаются.
- Полный обход всех 9248 клиник занимает ~10–15 минут.
